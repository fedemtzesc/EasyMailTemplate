📝 Documentación de Uso - Editor de Plantillas HTML Pro

Bienvenido al Editor de Plantillas. Esta herramienta está diseñada para configurar el contenido HTML de un correo electrónico y sus parámetros de envío, asegurando una correcta previsualización antes de su despliegue.

1. Gestión del Contenido HTML (Editor Izquierdo Inferior)

El área inferior del panel izquierdo es el corazón del editor. Aquí debe pegar o escribir su código HTML de correo electrónico. Es vital que el código esté optimizado para la compatibilidad con múltiples clientes de correo (tablas, CSS *inline*, etc.).

  • **Cargar HTML:** Utilice este botón para importar un archivo `.html` existente desde su equipo. El contenido se cargará automáticamente en el área de texto.
  • **Previsualización Dinámica:** Cualquier cambio que realice en el `textarea` se reflejará inmediatamente en el `iframe` de previsualización a la derecha.
2. Configuración de Datos (Panel Izquierdo Superior)

Esta sección define cómo se llamará la plantilla y cómo se tratarán las variables dinámicas en el contenido.

Nombre de Plantilla:

Es un campo crucial para identificar la plantilla en el sistema de envío. Debe ser único y descriptivo (ej. `promocion_navidad_2024`). Se recomienda utilizar minúsculas y guiones bajos (`_`) para separación.

JSON Request Ejemplo:

Defina aquí un objeto JSON que sirva como modelo de las variables que se inyectarán en su HTML. El sistema buscará variables dentro del HTML con la sintaxis `{{clave}}` y las reemplazará con el `valor` del JSON.

Ejemplo: Si en el HTML tiene `Hola {{nombre_cliente}}`, el JSON debe contener `{"nombre_cliente": "Juan Pérez"}`.

3. Manejo de Imágenes para Previsualización

Para asegurar que las imágenes se vean correctamente en el *preview* sin alterar el HTML de producción, siga estos pasos:

  1. **Uso Estándar del SRC:** En su código HTML, las imágenes deben usar la ruta estándar del servidor, que es: **`src="img/nombre_de_archivo.extension"`**.
  2. **Subir Imágenes:** Utilice el botón **"Subir imágenes"** y cargue los archivos de imagen correspondientes.
  3. **Mapeo Automático:** El sistema tomará el nombre del archivo subido (ej. `logo.png`) y lo mapeará a la ruta `img/logo.png` dentro del *preview*, utilizando una URL temporal Base64 para visualizarlas localmente.

**ATENCIÓN:** Si el nombre del archivo subido no coincide exactamente con el nombre de archivo en su `src`, la imagen NO se verá en la previsualización.

4. Configuración de Frecuencia y Límites

El selector **Frecuencia de Envío** determina el comportamiento de la plantilla en el motor de envíos:

  • **Enviar Inmediatamente:** La plantilla se marca para un envío único tan pronto como sea posible.
  • **Por fecha y hora de envío:** Permite especificar una fecha y hora (`datetime-local`) exacta para un envío programado y único en el futuro.
  • **Repetir todos los días a esta hora:** Define la hora (`time`) a la que se debe iniciar el proceso de envío diariamente. Esta opción requiere definir un límite.

Límite de Repetición (Solo para Envíos Diarios):

Define cuándo debe detenerse el envío diario:

  • **Sin límite (Continuo):** El envío continuará indefinidamente.
  • **Por cantidad de veces:** Detiene el envío después de que se haya ejecutado el número de veces especificado.
  • **Por fecha de término:** Detiene el envío automáticamente al alcanzar la fecha seleccionada.
5. Guardar y Despliegue

Una vez que todas las configuraciones estén completas, haga clic en **Guardar plantilla**. Esto compila todos los datos (HTML original, nombre, JSON y frecuencia) y los envía al servidor para ser almacenados y programados. El HTML que se guarda es el **original** (con `img/...`), no la versión con Base64.
